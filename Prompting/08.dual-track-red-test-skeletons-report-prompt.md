# UI RED Tests

Boundary / façade under test: the **public entry** that accepts `int[][]`, validates, then (on success) returns `int[6]`. Each test **must fail RED** until that boundary exists and enforces the contract. (Report/02 traceability: **T-U02, T-U03, T-U04/T-U05, T-U06/T-U07, T-U08, T-U09, T-U12**.)

---

**UI-RED-01** (maps **T-U02 / T-U03**)

| Field | Content |
|--------|---------|
| **Test name** | `test_ui_rejects_non_4x4_matrix_raises_or_structured_error` |
| **Given** | A matrix that is not 4×4 (e.g. `3×3` **or** `4` rows with one row `length != 4` / jagged). |
| **When** | The boundary `complete(...)` / `solve(...)` / equivalent public API is invoked. |
| **Then** | Failure before any domain solve: **`ValueError`** or project error with code **`E_WRONG_SIZE`** and message **`Matrix must be 4x4 with each row length 4.`** (per Report/02 §2.4). Domain must **not** be called if using a mock. |
| **Contract protected** | Input schema: outer length 4, each row non-null and length 4. |
| **Scenario** | Non-4×4 input is rejected at the boundary. |
| **Invariant protection** | **D1** — grid order is 4×4. |

---

**UI-RED-02** (maps **T-U04 / T-U05**)

| Field | Content |
|--------|---------|
| **Test name** | `test_ui_rejects_blank_count_not_two_raises_or_structured_error` |
| **Given** | Structurally 4×4 with **zero count ≠ 2** (e.g. 0, 1, or 3+ cells `0`), other cells in `0..16` for the case. |
| **When** | Boundary invoked. |
| **Then** | **`E_BLANK_COUNT`**, message **`Exactly two cells must be 0 (empty).`** |
| **Contract protected** | Exactly two blanks. |
| **Scenario** | Not exactly two blanks raises. |
| **Invariant protection** | **D2** — exactly two empty cells. |

---

**UI-RED-03** (maps **T-U06 / T-U07**)

| Field | Content |
|--------|---------|
| **Test name** | `test_ui_rejects_value_outside_zero_to_sixteen_raises_or_structured_error` |
| **Given** | 4×4 with two `0`s, but at least one cell **&lt; 0** or **&gt; 16** (e.g. `17` or `-1`). |
| **When** | Boundary invoked. |
| **Then** | **`E_VALUE_RANGE`**, message **`Each cell must be 0 or between 1 and 16 inclusive.`** Optional `details` for first violation in row-major order per Report/02 validation order. |
| **Contract protected** | Cell values only `0` or `1–16`. |
| **Scenario** | Value range violation raises. |
| **Invariant protection** | **D2** (value range legality). |

---

**UI-RED-04** (maps **T-U08**)

| Field | Content |
|--------|---------|
| **Test name** | `test_ui_rejects_duplicate_nonzero_raises_or_structured_error` |
| **Given** | 4×4, exactly two `0`s, all values in range, but **same non-zero value appears twice**. |
| **When** | Boundary invoked. |
| **Then** | **`E_DUPLICATE_VALUE`**, message **`Values 1 through 16 must not repeat except for 0.`** |
| **Contract protected** | No duplicate among non-zero cells. |
| **Scenario** | Duplicate numbers raise. |
| **Invariant protection** | **D3** — no duplicate non-zero values. |

---

**UI-RED-05** (maps **T-U09** length slice)

| Field | Content |
|--------|---------|
| **Test name** | `test_ui_success_returns_array_of_length_six` |
| **Given** | A **valid** partial grid per contract (4×4, two `0`s, 1–16 no dup), or stub domain returning a fixed `int[6]` if testing boundary mapping only. |
| **When** | Boundary completes successfully. |
| **Then** | `len(result) == 6`. |
| **Contract protected** | Output schema: `int[6]`. |
| **Scenario** | Returned array length must be 6. |
| **Invariant protection** | External **output shape** contract (Report/02 success schema); supports later **D4–D7** checks on indices 0–5. |

---

**UI-RED-06** (maps **T-U12**)

| Field | Content |
|--------|---------|
| **Test name** | `test_ui_success_coordinates_are_one_indexed` |
| **Given** | Valid input with blanks at known **1-based** positions (e.g. first blank `(2,3)` in 1-based terms), or stub that returns known coords. |
| **When** | Boundary returns `[r1,c1,n1,r2,c2,n2]`. |
| **Then** | `1 <= r1,c1,r2,c2 <= 4` and they match **row-major ordered blank cells** in 1-based indexing (**D7** mapping through boundary). |
| **Contract protected** | Coordinates are **1-indexed** and align with blank order. |
| **Scenario** | Returned coordinates must be 1-indexed. |
| **Invariant protection** | **D7** (blank order + 1-based external contract). |

# Logic RED Tests

Pure **domain** (or named helpers `find_blank_coords`, `find_not_exist_nums`, `is_magic_square`, `solution`) with **no** boundary implementation. Each test **fails RED** until the corresponding function/module exists and satisfies the invariant.

---

**LOGIC-RED-01 — `find_blank_coords()`**

| Field | Content |
|--------|---------|
| **Test name** | `test_find_blank_coords_returns_two_pairs_row_major_one_indexed` |
| **Given** | Valid partial: two `0`s at distinct positions (fixture: e.g. blanks at `(1,4)` and `(3,2)` in 1-based). |
| **When** | `find_blank_coords(grid)` |
| **Then** | Returns **exactly two** coordinate pairs; order is **row-major** (lexicographic on `(row,col)` 1-based); each row/col in `1..4`. |
| **Failing RED note** | Assert fails: missing API, wrong length, wrong order, or 0-based off-by-one. |
| **Scenario** | Return exactly 2 in row-major order. |
| **Invariant protection** | **D7** — blank order. |

---

**LOGIC-RED-02 — `find_not_exist_nums()`**

| Field | Content |
|--------|---------|
| **Test name** | `test_find_not_exist_nums_returns_two_missing_sorted_ascending` |
| **Given** | Valid partial with 14 distinct values from `1..16` and two missing (e.g. missing `7` and `12`). |
| **When** | `find_not_exist_nums(grid)` |
| **Then** | Returns the two absent values **`(a, b)` with `a < b`** (ascending). |
| **Failing RED note** | Assert fails until multiset complement of `{1..16}` is implemented. |
| **Scenario** | Two missing numbers, ascending order. |
| **Invariant protection** | **D4** (partial view: placed ∪ missing = `{1..16}`); pairs with **UC-D2** / `missingTwoNumbers` in Report/02. |

---

**LOGIC-RED-03 — `is_magic_square()`**

| Field | Content |
|--------|---------|
| **Test name** | `test_is_magic_square_true_when_all_sums_equal_thirty_four` |
| **Given** | Full 4×4, no zeros, classic 1–16 magic completion (all rows/cols/diagonals 34). |
| **When** | `is_magic_square(grid)` |
| **Then** | `True`. |
| **Failing RED note** | Fails until checker exists. |

| **Test name** | `test_is_magic_square_false_when_any_row_sum_not_thirty_four` |
| **Given** | Full grid with one row sum ≠ 34. |
| **When** | `is_magic_square(grid)` |
| **Then** | `False`. |

| **Test name** | `test_is_magic_square_false_when_any_column_sum_not_thirty_four` |
| **Given** | Full grid with one column sum ≠ 34. |
| **When** | `is_magic_square(grid)` |
| **Then** | `False`. |

| **Test name** | `test_is_magic_square_false_when_main_diagonal_sum_not_thirty_four` |
| **Given** | Full grid with main diagonal sum ≠ 34 (Report/02 **T-D09** style). |
| **When** | `is_magic_square(grid)` |
| **Then** | `False`. |

| **Test name** | `test_is_magic_square_false_when_anti_diagonal_sum_not_thirty_four` |
| **Given** | Full grid with anti-diagonal sum ≠ 34. |
| **When** | `is_magic_square(grid)` |
| **Then** | `False`. |

| **Scenario** | All row sums equal; all column sums equal; both diagonal sums equal; match **34**. |
| **Invariant protection** | **D5** — magic constant 34 on completed grid. |

---

**LOGIC-RED-04 — `solution()`**

| Field | Content |
|--------|---------|
| **Test name** | `test_solution_prefers_smaller_missing_at_first_blank_when_that_completion_is_magic` |
| **Given** | Well-posed fixture where assignment **smaller → first blank, larger → second** yields a completed magic square. |
| **When** | `solution(grid)` |
| **Then** | `[r1,c1,n1,r2,c2,n2]` has `(n1,n2) = (a,b)` with `a < b` per **D6** first branch. |

| **Test name** | `test_solution_swaps_numbers_when_only_reverse_assignment_is_magic` |
| **Given** | Fixture where only **larger → first blank, smaller → second** completes magic. |
| **When** | `solution(grid)` |
| **Then** | `(n1,n2) = (b,a)` (reversed relative to sorted missing pair). |

| **Test name** | `test_solution_output_length_six_and_coords_one_indexed` |
| **Given** | Any valid solvable partial. |
| **When** | `solution(grid)` |
| **Then** | `len == 6` and `1 <= r1,c1,r2,c2 <= 4`. |

| **Failing RED note** | All fail until `solution` wires **D7** blanks + **D6** try-order + completion check **D5**. |
| **Scenario** | Try small → first blank; on failure try reverse; return length 6; 1-indexed coordinates. |
| **Invariant protection** | **D6** (output number order), **D7** (blank order / coords), **D5** (magic check), **D4** on completed trial grid. |
