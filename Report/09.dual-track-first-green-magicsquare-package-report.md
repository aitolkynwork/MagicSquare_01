# Dual-track first GREEN — `magicsquare` package (Report 09)

**Scope of this document:** Meta record of the **first minimal GREEN** slice after [Report/08 — Dual-track RED test skeletons](08.dual-track-red-test-skeletons-report.md): **Track B** (domain `find_blank_coords`, **L-RED-01**) and **Track A** (boundary size/null validation, **U-RED-01**), plus the **`magicsquare`** Python package layout and tooling updates. **GREEN only** for this slice (no REFACTOR pass recorded here).

**Date:** 2026-04-28 (initial).

**Authoritative contracts & invariants:** [Report/02 — Layer design, contracts, TDD](02.layer-design-contracts-tdd-report.md) (**D1**, **D7**, error literals for boundary).

**Explicitly out of scope for this report:** Version-control or branching workflows; PyQt / screen layer; full boundary validation pipeline (value range, blank count, duplicates); domain magic checks, missing-number logic, or solve orchestration.

---

## 1. Task intent

1. Replace the placeholder **top-level `boundary/`** experiment with a single installable package **`magicsquare`** so **domain** and **boundary** live under one namespace aligned with the **`pyproject.toml`** project name.
2. Introduce a shared **`MATRIX_SIZE`** constant so grid dimension is not scattered as a magic literal in loops (contract error strings remain the fixed literals from [README.md](../README.md) / Report/02).
3. Land **minimal** production code for exactly two pytest greens: **LOGIC-RED-01** and **UI-RED-01**, keeping remaining RED stubs unchanged.

## 3. Track B — Logic GREEN (L-RED-01)

| Item | Detail |
|------|--------|
| **Test** | `test_find_blank_coords_returns_two_pairs_row_major_one_indexed` |
| **API** | `find_blank_coords(matrix: Sequence[Sequence[int]]) -> list[tuple[int, int]]` |
| **Rule** | Scan `r, c` in `0 .. MATRIX_SIZE-1` row-major; emit `(r+1, c+1)` for each `matrix[r][c] == 0`. |
| **Invariant** | **D7** — blank order for output coordination; caller assumed to provide a valid partial grid for this minimal test. |

## 4. Track A — UI / boundary GREEN (U-RED-01)

| Item | Detail |
|------|--------|
| **Test** | `test_ui_rejects_non_4x4_matrix_raises_or_structured_error` |
| **API** | `validate_ui_matrix(matrix)` — accepts `None` or nested rows. |
