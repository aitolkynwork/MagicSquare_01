# Refactoring plan and smell analysis report (Report 11)

**Scope of this document:** Consolidate the current code smell review and ECB/SRP analysis for `magicsquare/domain.py`, `magicsquare/boundary.py`, and `magicsquare/gui/window.py` (requested as `gui/main_window.py`), then define a test-first refactoring sequence.

**Date:** 2026-04-28

---

## 1. Files reviewed

- `magicsquare/domain.py`
- `magicsquare/boundary.py`
- `magicsquare/gui/window.py` (no `magicsquare/gui/main_window.py` file found in repository)

---

## 2. Key findings (ECB + SRP + maintainability)

| File | Finding category | Observation | Risk |
|------|------------------|-------------|------|
| `magicsquare/domain.py` | ECB layering | Mixed role: pure entity rules plus orchestration in `complete_magic_square`. | Control flow is coupled to entity utilities, reducing layering clarity. |
| `magicsquare/domain.py` | SRP | `complete_magic_square` performs multiple responsibilities (collect inputs, run trials, evaluate outcomes, choose result/raise). | Harder to reason about and refactor safely. |
| `magicsquare/domain.py` | Naming | `find_not_exist_nums`, short temporaries (`a`, `b`, `br1`, `bc1`) reduce intent clarity. | Slower code comprehension and review friction. |
| `magicsquare/boundary.py` | SRP | `validate_ui_matrix` combines multiple validation stages in one long function. | Contract changes likely to produce regressions. |
| `magicsquare/boundary.py` | Duplication | Size error construction/message repeated in multiple checks. | Inconsistent edits likely during future changes. |
| `magicsquare/boundary.py` | Dead field | `UiBoundaryError.details` is assigned but currently not used in project reads. | Potential confusion and unused API surface. |
| `magicsquare/gui/window.py` | UI maintainability | `MainWindow.__init__` is a long setup method. | UI changes become harder to isolate and test. |
| `magicsquare/gui/window.py` | Magic numbers | `setRange(0, 16)` encodes domain limits in UI layer. | Limits can drift from shared constants. |

---

## 3. ECB role assessment

| File | Current role | Assessment |
|------|--------------|------------|
| `magicsquare/domain.py` | Entity + Control mixed | Should be split so entity keeps pure rules and control owns solve orchestration. |
| `magicsquare/boundary.py` | Boundary | Role is correct; internal decomposition can improve SRP without changing boundary contract. |
| `magicsquare/gui/window.py` | Boundary (UI) | Mostly correct UI boundary; no direct business judgment like `total == 34` in UI code. |

---

## 4. Refactoring targets (priority order)

| # | Target file | Issue | Technique | Priority |
|---|-------------|-------|-----------|----------|
| 1 | `magicsquare/domain.py` | `complete_magic_square` mixes entity rule helpers and use-case orchestration | Extract use-case flow into Control module; keep pure entity helpers in domain | High |
| 2 | `magicsquare/boundary.py` | `validate_ui_matrix` long multi-stage validator | Extract validation sub-functions per rule group while preserving error codes/messages | High |
| 3 | `magicsquare/boundary.py` | `solve_ui_matrix` contains mapping and dispatch branching | Extract exception-translation helper; keep facade API unchanged | Medium |
| 4 | `magicsquare/domain.py` | Hard-to-understand names | Rename for intent clarity with characterization tests guarding behavior | Medium |
| 5 | `magicsquare/gui/window.py` | Long constructor and literal range values | Extract UI builder methods; replace literal limits with shared constants | Medium |
| 6 | `magicsquare/boundary.py` | Repeated size error construction | Introduce local helper/constant for repeated payload | Low |

## 6. Post-refactor verification

- Regression command:

```text
pytest -q
```

- External behavior unchanged criteria:
  - Existing tests pass with no expectation updates for current contracts.
  - Boundary error `code` and `message` values remain exactly the same.
  - Solver output contract remains `[r1, c1, n1, r2, c2, n2]` in current order.
  - UI behavior remains unchanged for success and failure flows (result label and warning dialog behavior).
