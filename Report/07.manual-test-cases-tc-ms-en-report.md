# Manual test cases TC-MS (English) — Report 07

**Scope of this document:** Meta summary of **[docs/test-cases-TC-MS-english.md](../docs/test-cases-TC-MS-english.md)** — **22** procedural test cases (**TC-MS-A** through **TC-MS-D**) translated into English and laid out in a **vertical specification template** (project header, per-case purpose, function under test, input, Given/When/Then steps, environment, prerequisites, pass/fail criteria, special procedures, priority).  
**Excluded by intent:** Full duplication of the test-case body — use `docs/test-cases-TC-MS-english.md` as the working artifact; Report/02 remains the authoritative **automated** test IDs (**T-D\***, **T-U\***, etc.).

**Date:** 2026-04-28 (initial).

---

## 1. Task intent

1. Capture **manual / QA-style** scenarios (origin: Korean test-case table; functions named `find_blank_coords`, `is_magic_square`, `solution` in that source) in **English** for onboarding, audits, or **non-pytest** walkthroughs.
2. Provide a **repeatable document template** (header rows + per-test block) suitable for spreadsheets or test-management tools.
3. **Map** each group to MagicSquare_01 concepts where they overlap (**D1–D7**, **E\_\***, Report/02 **T-\*** family) and call out **gaps** (e.g. blank-count variants **A-003–A-005** vs strict “exactly two blanks” in the product contract).

## 4. Naming and API alignment

- The English doc uses **placeholder function names** from the source table (`find_blank_coords`, `is_magic_square`, `solution`). Implementation names may follow ECB (**entity** / **control** / **boundary**) per [Report/02](02.layer-design-contracts-tdd-report.md) and [`.cursorrules`](../.cursorrules); when automating, **map** TC-MS rows to **T-D\***, **T-U\***, **T-I\*** tests rather than renaming production code to match this doc.
- **Indexing:** the source mixed **0-based** and **1-based** coordinates; `docs/test-cases-TC-MS-english.md` instructs aligners to use project rules (**D7** row-major, **1-based** in product I/O).
- **Exceptions vs booleans:** Group **D** cites `ValueError`, `TypeError`, or `False` per original spec; MagicSquare_01 may surface **structured errors** (**E\_\***) — the doc notes reconciliation where needed.
