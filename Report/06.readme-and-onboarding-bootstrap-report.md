## 1. Task intent

Provide a single **repository entry point** so contributors and tools (including Cursor) can:

1. Anchor product understanding on **[docs/MagicSquare-4x4-Completion-System-Report.md](../docs/MagicSquare-4x4-Completion-System-Report.md)**.
2. Reuse **story and epic wording** from [Report/04.user-journey-levels-1-5-report.md](04.user-journey-levels-1-5-report.md).
3. Reuse **external contracts, error codes, literal messages, and validation order** from [Report/02.layer-design-contracts-tdd-report.md](02.layer-design-contracts-tdd-report.md).
4. Reuse **execution posture** (ECB, RED–GREEN–REFACTOR, pytest, coverage floor, layout) from [Report/03.cursorrules-configuration-report.md](03.cursorrules-configuration-report.md), the authoritative [`.cursorrules`](../.cursorrules), and **`pyproject.toml`**.
5. Record **one-line document relationships** consistent with [Report/05.making-pdr-report.md](05.making-pdr-report.md) (what each artifact is for, without replacing canonical bodies).
6. Embed an **implementation To-Do** (**Epic-001**, **US-001–006**, **TASK-000–028**) with **sequence thinking**, **RED / GREEN / REFACTOR** micro-steps where instructive, and a **requirements traceability matrix** (**Task → Req → Scenario → Test → Status**) so the README behaves like a Cursor-oriented practice dev board.

## 3. README “Implementation To-Do” structure (summary)

The To-Do section is titled **Implementation To-Do (checklist, sequence thinking, traceability)** and includes:

| Block | Purpose |
|-------|---------|
| **Sequence thinking** | (1) Vertical order: entity → control → boundary → integration/gates. (2) Per-task TDD: RED → GREEN → REFACTOR per `.cursorrules`. (3) Horizontal link: **Task → Req → Scenario → Test** (concept-to-code / C2C-style). |
| **Recommended task order** | Explicit ID sequence `TASK-000` … `TASK-028`. |
| **Epic-001** | Checkbox plus **done** definition tying together US-002–004 domain path, US-001/US-005 path, integration **TASK-023–025**, coverage **TASK-027**; US-006 optional. |
| **US headings** | Tasks nested under **US-002** … **US-005**, **US-001**, **US-006**, then integration — **implementation sequence**, not numeric US order. |
| **Micro-steps** | Selected tasks decomposed into **TASK-nnn-1** (RED), **-2** (GREEN), **-3** (REFACTOR); **TASK-008** adds an extra RED for **T-D09**. |
| **Traceability matrix** | Columns: Task ID, Req ID, Scenario (L0/L1/L2/L3), Test, Status (⬜ TODO placeholder; team may use 🔴 RED / ✅ PASS). |
| **User story index** | One-line summaries of **US-001–006** for backlog tools. |

Canonical test IDs (**T-D\***, **T-U\***, **T-I\***, **T-DS\***) remain defined in Report/02; the matrix and checklist **reference** them by name or ID.

- When **Report/02** contracts (I/O, messages, test IDs) change, update **README.md** contract tables, the **traceability matrix** rows, and any **TASK-xxx** / **T-D\*/T-U\*** references.
